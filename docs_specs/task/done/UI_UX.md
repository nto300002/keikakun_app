## 2. ダッシュボードの空状態とMFA未設定説明

対象画面:

- ダッシュボード
- 利用者が0件の状態
- 2段階認証未設定で利用者の作成・編集が制限される状態

修正箇所:

- `k_front/components/protected/dashboard/Dashboard.tsx`
- 関連: `k_front/components/billing/BillingProtectedButton.tsx`
- 関連: `k_front/lib/billing/status.ts`

受け入れ要件:

- [x] 利用者が0件のとき、空白ではなく「まだ利用者が登録されていません」趣旨の空状態を表示する。
- [x] オーナー/マネージャーで `staff.is_mfa_enabled = true` かつ課金状態が書き込み可能な場合、空状態に「利用者を登録する」導線を表示する。
- [x] `staff.is_mfa_enabled = false` の場合、利用者作成ボタンを非表示または無効化し、2段階認証の設定が必要であることを明示する。
- [x] MFA未設定時の説明は「セキュリティのため、2段階認証を設定すると利用者の新規作成・編集ができます」趣旨にする。
- [x] 課金制限とMFA制限が同時にある場合、原因が混ざらないよう、現在の警告表示と矛盾しない。
- [x] employee権限など、もともと作成権限がないユーザーには誤って作成導線を出さない。

確認方法:

- [ ] 利用者0件 + MFA有効 + 書き込み可能な課金状態で、空状態と作成導線を確認する。
- [ ] 利用者0件 + MFA無効で、2段階認証設定が必要な説明を確認する。
- [ ] 利用者あり + MFA無効で、既存のカード表示と操作制限説明が崩れないことを確認する。
- [ ] `trial_expired` / `payment_failed` / `canceled` の課金制限表示とMFA説明が競合しないことを確認する。
