
## 7. チュートリアルボタン
- ヘッダー
利用者ダッシュボードについて、更新期限がどこにあるか、アセスメントと個別支援計画ページへはどこから遷移できるか
- 管理者メニュー
管理者メニューで何ができるか説明する
> 事務所スタッフ管理(mfa)、退会処理について、GoogleCalendar連携について

チュートリアル
コーチマーク / Coach marks（ガイド枠＋矢印）

実装方法（技術的ヒント）

単純オーバーレイ：半透明の <div>（position: fixed; inset:0; background: rgba(0,0,0,0.6)）を重ねる。
穴あきスポットライト：SVG マスクか CSS mask/clip-path を使い「穴」を作る（高精度・角丸対応）。
高速で簡単：対象外を暗くするオーバーレイ＋要素に高 z-index の枠/吹き出しを置く。
pointer-events：オーバーレイで背景操作を防ぎつつ、スポット領域は pointer-events: none にして操作可能にする（要注意）。
アニメーション：フェード・ズームで注目導線を付けるが、減速設定（prefers-reduced-motion）を尊重する。
よく使われるライブラリ（導入で実装が楽）

react-joyride（React 向けのガイドツアー）
reactour / driver.js（スポットライト型のツアー）
Shepherd.js / intro.js（多機能でブラウザ対応良好）
tippy.js（ツールチップ／コーチマーク用）
アクセシビリティとUXのチェックポイント

キーボード操作で次/前/閉じるができる（Tab、Enter、Esc を扱う）
フォーカス管理：チュートリアル中はフォーカスをツアーに移し、終了時に元要素へ戻す
スクリーンリーダー：aria-describedby / aria-hidden を適切に設定、ステップ変更は aria-live で通知
いつでもスキップ／閉じられるようにする（強制不可）
モバイルでの表示：スポットの位置が小さすぎない、スクロールで対象が隠れないようスクロール誘導
ローカライズ：日本語で簡潔に、操作指示は動詞で明示（例：「ここをクリックして設定を開く」）
簡単な文言例（日本語）

「ここからアセスメントに移動できます」
「更新期限が近い利用者はここに表示されます」
ボタンラベル: 「次へ」「前へ」「スキップ」「閉じる」
